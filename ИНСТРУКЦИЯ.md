# 🤖 CloseCLAW — Инструкция по запуску

> **AI Bot Commander** — Телеграм-бот, который управляет Linux-сервером, запускает дочерние боты и общается с двумя ИИ: облачным (io.net) и локальным (Ollama).

---

## 📁 Шаг 1 — Открыть папку проекта

```
C:\AG\CloseCLAW\
```

Всё что есть в папке:

| Файл / Папка | Что это |
|---|---|
| `.env` | **Главный конфиг** — сюда прописываются все ключи |
| `bot.py` | Точка входа — основной бот |

| `dashboard/` | Веб-дашборд (открывается в браузере) |
| `requirements.txt` | Python-зависимости |

## 🔑 Шаг 2 — Прописать API ключи в `.env`

Открыть файл `.env` (он в корне папки) и заполнить все поля:

```env
# ── ОБЯЗАТЕЛЬНО ──────────────────────────────────────────

# Токен мастер-бота от @BotFather в Телеграме
TELEGRAM_BOT_TOKEN=your_master_bot_token_here

# API ключ от io.net (облачный AI)
# Получить: https://cloud.io.net → API Keys
IO_NET_API_KEY=your_ionet_api_key_here

# Твой Telegram User ID (узнать у @userinfobot)
# Можно несколько через запятую: 123456,789012
ALLOWED_USER_IDS=123456789,987654321

# Пароль для шифрования хранилища токенов (придумай сам, любой)
TOKEN_STORE_PASSWORD=change_this_to_strong_password

# ── НЕОБЯЗАТЕЛЬНО (менять только если надо) ──────────────

# Адрес локального Ollama (если запущен на этой же машине — не трогать)
OLLAMA_BASE_URL=http://localhost:11434

# Модель Ollama для локального AI
OLLAMA_MODEL=llama3.2

# Базовый URL io.net API (не трогать)
IONET_BASE_URL=https://api.intelligence.io.solutions/api/v1

# Модель io.net (можно поменять на другую)
IONET_MODEL=meta-llama/Llama-3.3-70B-Instruct

# Порт веб-дашборда
DASHBOARD_PORT=3000
```

### Где взять ключи?

| Ключ | Где получить |
|---|---|
| `TELEGRAM_BOT_TOKEN` | Написать [@BotFather](https://t.me/BotFather) → `/newbot` → скопировать токен |
| `IO_NET_API_KEY` | [cloud.io.net](https://cloud.io.net) → войти → раздел **API Keys** → создать новый |
| `ALLOWED_USER_IDS` | Написать боту [@userinfobot](https://t.me/userinfobot) — он пришлёт твой ID |

## 🐍 Шаг 3 — Установить Python-зависимости

> **Требуется:** Python 3.10+

Открыть терминал в папке `C:\AG\CloseCLAW\` и выполнить:

```bash
# Создать виртуальное окружение (один раз)
python -m venv .venv

# Активировать окружение
# На Windows:
.venv\Scripts\activate
# На Linux/Mac:
source .venv/bin/activate

# Установить все зависимости
pip install -r requirements.txt
```

## 🦙 Шаг 4 — Запустить Ollama (локальный AI, опционально)

> Если хочешь использовать команду `/local` — нужен Ollama.  
> Если нет — пропусти этот шаг, облачный `/ai` работает без него.

```bash
# Установить Ollama: https://ollama.com/download

# Запустить нужную модель (первый раз — скачает автоматически)
ollama run llama3.2
```

## 📟 Команды бота

| Команда | Что делает |
|---|---|
| `/start` | Показать список всех команд |
| `/status` | CPU, RAM, диск, список запущенных ботов |
| `/run <команда>` | Выполнить shell-команду на сервере |
| `/ai <вопрос>` | Спросить облачный AI (io.net) |
| `/local <вопрос>` | Спросить локальный AI (Ollama, приватно) |
| `/createbot <описание>` | Создать и запустить нового Telegram-бота |
| `/bots` | Список всех запущенных дочерних ботов |
| `/stopbot <имя>` | Остановить дочерний бот |
| `/tokens` | Показать список сохранённых токенов |
| `/addtoken <имя> <токен>` | Сохранить токен бота (шифруется AES-256) |
| `/removetoken <имя>` | Удалить токен |

> 💡 **Умный роутинг:** можно просто написать что угодно без команды — бот сам решит, что делать и куда отправить запрос.

## ❗ Частые ошибки

| Ошибка | Решение |
|---|---|
| `TELEGRAM_BOT_TOKEN not set` | Не заполнен `.env` файл |
| `Access denied` в боте | Твой ID не добавлен в `ALLOWED_USER_IDS` |
| `Ollama error` | Ollama не запущен, или модель не скачана |
| `io.net error` | Неверный `IO_NET_API_KEY` или нет интернета |
| `ModuleNotFoundError` | Не установлены зависимости / не активировано `.venv` |

---

## 🔒 Безопасность

- Файл `.env` **никогда не заливать в git** (уже добавлен в `.gitignore`)
- `TOKEN_STORE_PASSWORD` — придумай надёжный пароль, он шифрует хранилище токенов
- `ALLOWED_USER_IDS` — добавляй только свои ID, чтобы никто чужой не получил доступ к серверу
