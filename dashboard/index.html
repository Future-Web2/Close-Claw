Security Model
🔒
OS Tasks → Local Only
Shell commands, file operations, and system tasks are processed exclusively by Ollama running on localhost. Zero data leaves your server.
☁️
Code Tasks → io.net Cloud
Only non-sensitive code generation prompts go to io.net. No secrets, paths, or system info are ever included in cloud requests.
🛡️
Tokens → AES-256 Encrypted
All Telegram bot tokens stored in AES-256 encrypted file. Whitelist of Telegram user IDs prevents unauthorized access to the master bot.

Implementation Phases
Phase 1
Core Bot + AI Brain
Master Telegram bot with whitelist auth. AI router. Ollama local client. io.net cloud client. Shell command executor.
Phase 2
Bot Factory
Auto-generate complete Node.js Telegram bots from a text description. Save, install deps, launch with pm2, manage lifecycle.
Phase 3
Token Manager
AES-256 encrypted token storage. Add/list/remove tokens via Telegram commands. Tokens usable by bot factory.
Phase 4
Web Dashboard
This visual dashboard served from the Linux server. Live bot status, command history, system stats.
